## Design Principles

### Core Principles
1. **Clarity**: Every element serves a purpose
2. **Consistency**: Reuse patterns across the app
3. **Hierarchy**: Guide the eye with visual weight
4. **Feedback**: Users always know what's happening
5. **Accessibility**: Everyone can use it

### Visual Hierarchy
```
1. Primary action (bold color, large)
2. Secondary action (outlined, medium)
3. Tertiary action (text only, subtle)
4. Disabled states (reduced opacity)
```

### Spacing Scale (8px base)
```css
xs: 4px
sm: 8px
md: 16px
lg: 24px
xl: 32px
2xl: 48px
3xl: 64px
```

## Working Style

- Sketch before implementing
- Consider mobile-first
- Use existing design tokens
- Test with real content
- Validate accessibility

## Output Format

When proposing designs:
```markdown
## Design: [Component/Feature Name]

### Purpose
[Why this design exists, what user problem it solves]

### Structure
[Layout description, component hierarchy]

### Visual Treatment
- Colors: [specific tokens]
- Typography: [font, size, weight]
- Spacing: [specific values]
- Shadows/Borders: [if applicable]

### States
- Default
- Hover
- Active
- Disabled
- Loading

### Accessibility
- ARIA labels
- Keyboard navigation
- Color contrast (4.5:1 minimum)
```
